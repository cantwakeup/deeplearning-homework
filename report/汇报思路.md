# 三个实验汇报思路

## 总体开场

本次作业按任务书分为三个实验：纯 NumPy MLP、PyTorch CNN、PyTorch ResNet。汇报主线建议放在“从底层训练机制到卷积图像模型，再到深层残差网络”的递进关系上：

1. MLP 说明自己实现前向传播、损失函数、反向传播和参数更新。
2. CNN 说明卷积、池化、BatchNorm、Dropout 如何提升图像分类。
3. ResNet 说明残差连接如何支撑深层网络训练。
4. 最后强调：三个实验不只跑了单个结果，还做了系统的多参数搜索和结构消融，结果都保存在 `outputs/` 和 `latex_report/`。

## 实验一：NumPy MLP

汇报顺序：

1. 先说明限制条件：不能使用 PyTorch 的 `Linear`、`sigmoid` 等模块，权重、偏置、激活函数、Softmax、交叉熵和梯度更新都由 NumPy 实现。
2. 说明数据集：三分类螺旋数据，输入是二维坐标，输出是 3 类。该数据非线性可分，适合展示隐含层非线性表达能力。
3. 说明基础网络：输入层 2 维、单隐含层、输出层 3 类。
4. 说明扩展实验：比较 ReLU/Tanh/Sigmoid、隐藏层宽度 16/32/64/128、学习率 0.1/0.05/0.01/0.005、GD/Momentum/Adam，并对 Top 配置做 3 个随机种子复验。
5. 展示结果：基线测试准确率 0.9667；扩展实验最佳为 ReLU + hidden_dim=16 + Adam + lr=0.01，多种子平均测试准确率 0.9917，标准差约 0.0068。
6. 展示图：`mlp_optimizer_lr.png`、`mlp_activation_hidden.png`、`mlp_decision_boundary.png`。

可回答的问题：

- 为什么不用线性模型：螺旋数据非线性可分，需要隐含层和激活函数提供非线性表达。
- 为什么 Adam 表现好：它对不同参数自适应调整步长，通常比普通 GD 更快收敛。
- 为什么要多随机种子复验：避免只根据一次随机初始化得出偶然结论。

## 实验二：CNN 识别 MNIST

汇报顺序：

1. 先说明任务：使用 `torchvision.datasets.MNIST` 完成 10 分类，满足任务书“使用 torch 自带样本数据集”的要求。
2. 说明模型结构：两组卷积块提取局部笔画特征，分类器输出 10 类。
3. 说明第一组多情况分析：Adam/SGD/RMSprop 与多档学习率，共 9 组，结果保存在 `02_mnist_cnn/outputs/cnn_sweep/`。
4. 说明第二组结构消融：比较 3x3/5x5 卷积核、16/32/64/128 通道宽度、BatchNorm 有无、Dropout 0/0.25/0.5、池化有无，结果保存在 `02_mnist_cnn/outputs/cnn_arch_sweep/`。
5. 展示结果：CNN 单次基线测试准确率 0.9900；优化器/学习率 sweep 中 RMSprop + 0.0001 为 0.9896；架构消融中 5x5 卷积核配置达到 0.9915。
6. 展示图：`cnn_optimizer_lr_heatmap.png`、`cnn_arch_ablation.png`。

可回答的问题：

- 为什么 CNN 比 MLP 更适合图像：卷积通过局部连接和参数共享提取边缘、笔画等局部模式。
- 为什么 5x5 在本次结果中略好：MNIST 笔画较粗，稍大的感受野可以一次覆盖更多局部形状；但差距不大，说明任务本身较简单。
- 为什么没有池化变差：去掉池化后特征维度和参数量增加，短训练周期内优化更困难，也更容易过拟合。
- 为什么不同优化器差距不大：MNIST 简单，3 个 epoch 内多数合理配置都能达到高准确率，差异主要体现在收敛速度和学习率敏感性。

## 实验三：ResNet-MNIST

汇报顺序：

1. 先说明任务：按任务书建议实现 ResNet-50，并在 MNIST 上训练和预测。
2. 说明核心结构：ResNet-50 使用 Bottleneck，ResNet-18/34 使用 BasicBlock；残差连接将输入直接加到输出，缓解深层网络训练退化。
3. 说明 MNIST 适配：MNIST 是 1 通道 28x28 图像，因此默认使用 3x3、stride=1 的 MNIST stem；同时加入 ImageNet 风格 stem 作为对照。
4. 说明扩展实验：比较 ResNet-18/34/50、Plain-18、MNIST stem/ImageNet stem、Adam/SGD/RMSprop、不同学习率，结果保存在 `03_resnet_mnist/outputs/resnet_sweep/`。
5. 展示结果：ResNet-50 基线测试准确率 0.9872；扩展 sweep 中 ResNet-34 最好，测试准确率 0.9931；Plain-18 只有 0.9781，说明残差连接对稳定训练有明显帮助。
6. 展示图：`resnet_ablation.png`、`resnet_params_accuracy.png`。

可回答的问题：

- 为什么 ResNet-34 比 ResNet-50 高：MNIST 较简单，过深模型不一定更优；ResNet-50 参数更多，3 epoch 内优化成本更高。
- 为什么 ImageNet stem 不如 MNIST stem：ImageNet stem 早期下采样太强，MNIST 输入只有 28x28，过早压缩会损失笔画细节。
- 为什么要加入 Plain-18：它是残差连接的直接对照，能说明准确率提升来自结构设计，而不是单纯来自参数量。
